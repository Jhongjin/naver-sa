# Protected Test Execution Route

Date: 2026-05-03

## Scope

Added a protected route that can turn approved execution drafts into Naver Search Ad API requests.

Route:

- `POST /api/naver/execute-draft`

Modes:

- default: dry-run only, returns generated draft
- `execute: true`: attempts Naver API calls only when all safeguards pass

## Required Safeguards

Actual execution requires:

- `x-admin-secret` header matching `CRON_SECRET`
- request body `execute: true`
- request body `confirmation: "TEST_EXECUTION_ONLY"`
- approved staged changes
- configured Naver API environment variables

## Safety

- No browser UI calls this route.
- The route is disabled if `CRON_SECRET` is missing.
- The route stops on the first Naver API failure.
- Live activation and delete actions are not generated by the draft layer.
- Any real call still requires a separate human confirmation before we invoke it.

## Remaining Work

Before use against a real test account:

- pull or configure local environment variables safely
- confirm test-account-only execution scope
- resolve real channel/ad group IDs
- store pre/post execution audit events
